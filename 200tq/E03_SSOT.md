## Part 2: E03 시스템 정의

### 2.1 핵심 시그널

```
Signal = Majority Vote of [MA160, MA165, MA170] vs MA3

IF ≥2 windows are ON → Signal = ON
ELSE → Signal = OFF
```

- **ON 조건**: QQQ의 `SMA(3) > SMA(window)`
- **OFF 조건**: QQQ의 `SMA(3) ≤ SMA(window)`

### 2.2 포지션 배분

| 시그널  | TQQQ | SGOV |
| :------ | ---: | ---: |
| **ON**  | 100% |   0% |
| **OFF** |  10% |  90% |

### 2.3 실행 모델

- **체크 주기**: Daily (월간도 동등 성과 — E26 검증)
- **실행 지연**: 1일 (t일 시그널 → t+1일 적용)
- **리밸런싱**: Delta 방식 (목표 비중 변경 시에만)

## Part 4: 운영 레이어

### 4.1 Deadband Layer — ⏭️ E03 비적용

> [!IMPORTANT]
> **E03에서 Deadband는 적용하지 않습니다.**
>
> E03은 ON↔OFF 전환 시 **90%p 점프** (100% ↔ 10%)가 발생하므로,
> 3~5%p Deadband 임계값이 트리거될 상황이 없습니다.
> 연간 거래 횟수도 3.3회로 이미 극히 낮아 추가 필터링이 불필요합니다.
>
> **검증**: E30(5%p), E31(3%p) 실험 결과 E03 대비 CAGR/MDD/거래횟수 모두 동일.
> 상세: [deadband_verdict.md](./artifacts/off10_robustness/notes/deadband_verdict.md)

### 4.2 비상 전환 (Emergency Exit) 규칙

> [!CAUTION]
> **Flash Crash 대응 프로토콜**

| 트리거 조건                          | 행동                | 비고                                          |
| :----------------------------------- | :------------------ | :-------------------------------------------- |
| QQQ 당일 종가 기준 **-7% 이상** 하락 | **OFF10 강제 전환** | 다음 장 시작 시 TQQQ 10%, SGOV 90%로 리밸런싱 |
| TQQQ 진입가 대비 **-20% 이상** 하락  | **OFF10 강제 전환** | 동일                                          |

**핵심 원칙**:

- 비상 전환의 목표 상태는 **OFF10 (10% 잔류)**이며, 0% 청산이 아닙니다.
- 10% 잔류는 급반등 포착을 위한 핵심 설계이므로 비상 시에도 유지합니다.
- 자동 매매 시스템이 없는 경우, 대시보드 경보 확인 후 **수동 매도**합니다.

## Part 5: 실제 운용 체크리스트

### 5.1 Daily Ops (권장)

```
1. 장 마감 후 QQQ 종가 확인
2. MA3, MA160, MA165, MA170 계산
3. 다수결 투표 → ON/OFF 결정
4. 목표 비중 계산 (ON: 100/0, OFF: 10/90)
5. 전일 대비 시그널 변경 여부 확인 → 거래 필요 여부 판단
6. 다음 장 시작 시 실행
```

### 5.2 수동 매매 실행 프로토콜 (삼성증권 기준)

| 항목                 | 규칙                                                             |
| :------------------- | :--------------------------------------------------------------- |
| **시그널 확정 시점** | 미국 정규장 마감 후 (한국시간 익일 06:00~07:00)                  |
| **주문 입력 시점**   | 한국시간 22:30 (프리마켓) 또는 23:30 (정규장 시작)               |
| **주문 타입**        | **시장가 (MOO: Market on Open)** 권장                            |
| **10% 잔류 계산**    | **수량 기준**, 소수점 이하 **올림** (예: 보유 100주 → 10주 잔류) |
| **SGOV 매수**        | TQQQ 매도 체결 후, 체결금액의 90%를 시장가 매수                  |

> [!TIP]
> **10% 계산 예시**: TQQQ 137주 보유 시
>
> - 10% = 13.7주 → **14주 잔류** (올림)
> - 매도 수량 = 137 - 14 = **123주**

### 5.3 예외 상황

| 상황           | 대응                         |
| :------------- | :--------------------------- |
| 데이터 누락    | 전일 시그널 유지             |
| SGOV 거래 불가 | SHV로 대체                   |
| Flash Crash    | Part 4.2 비상 전환 규칙 적용 |

### 5.4 신규 자금 / 분배금 처리

| 상황               | 규칙                                                    |
| :----------------- | :------------------------------------------------------ |
| **신규 입금**      | 현재 시그널의 목표 비중에 맞춰 **부족한 자산**부터 매수 |
| **분배금 수령**    | 당일 시그널 확인 후, 목표 비중에 맞춰 재투자            |
| **최소 주문 미달** | SGOV에 현금 축적 후, 다음 리밸런싱 시점에 통합 처리     |
| **정수주 제약**    | 소수점 미만은 SGOV 쪽에 잔액으로 유지                   |
